  • ורמז לקדימה: בהמשך (מפגש 4) נראה שאפשר להפעיל מבחוץ — בלי לפתוח טרמינל בכלל

🔧 סקילים, CLAUDE.md וריצה רצופה
קונספט הדגמה

תזכורת מהירה על הבסיס — ועוד משהו חדש חשוב.

  • למה סקילים חשובים? דיוק מצד אחד, שירשור פעולות מצד שני

🆕 חדש: ריצה רצופה ללא הפסקות

הפקודה --dangerously-skip-permissions מאפשרת לסוכן לרוץ בלי לבקש אישור על כל פעולה. שולחים משימה — הולכים לעשות משהו אחר — חוזרים לתוצאות.

claude --dangerously-skip-permissions

⚠️ גם אחרי שמרשים הכל — יש דרכים להגביל פעולות ספציפיות. נגיע לזה בהמשך.

תת-סוכנים — איך הם עובדים באמת

תת-סוכן הוא סוכן שמופעל ע"י סוכן אחר. שלושה דברים שחשוב להבין:

רץ במקביל

🔒
כלים מוגבלים
אפשר לתחם בדיוק אילו כלים מותרים לכל תת-סוכן — קריאה בלבד, כתיבה, דפדפן וכו׳

🌐 כלי הדפדפן — Chrome Extension MCP
קונספט הדגמה

MCP = פרוטוקול חיבור בין הסוכן לכלים חיצוניים. התוסף לכרום הוא כלי MCP שנותן לסוכן גישה לדפדפן — הדפדפן שלכם, עם כל החשבונות שלכם.

🔐
זוכר סשנים
מחובר לחשבונות שלכם כבר. לא צריך להתחבר מחדש בכל פעם.
🗂️
מעבר טאבים
יכול לבצע משימות שדורשות כמה טאבים — למשל אימות מייל תוך כדי הרשמה.
🖼️
העלאת תמונות
יכול לפרסם ברשתות חברתיות כולל העלאת תמונה — פוסט שלם, בלי לגעת.
🎬
הקלטת GIF
מתעד כל מה שעשה — מה שמאפשר לשתף, לתעד, ולאמת שהכל עבד.
⚠️ הסוכן בדפדפן — איטי, אבל זה לא הנקודה

עבודה של סוכן בדפדפן לוקחת זמן — פי כמה יותר ממה שהייתם עושים ידנית. אבל השאלה הנכונה היא לא "כמה זמן זה לוקח?" אלא "האם הוא מסוגל לרוץ מקצה לקצה?"
אם כן — הוא יכול לעשות את זה בזמן שאתם לא נוכחים. בלילה. בסוף שבוע.

דוגמאות למשימות דפדפן שכדאי לתת לסוכן לילה:
📋 מילוי טפסים — הגשת מסמכים, הרשמות, עדכון פרטים במערכות
🔍 סריקת מידע — איסוף נתונים מאתרים, השוואת מחירים, מחקר שוק
📱 פרסום תוכן — העלאת פוסטים לרשתות חברתיות כולל תמונה וכיתוב
📧 תהליכי הרשמה — יצירת חשבונות, אימות מייל, מילוי פרופיל
✅ השיטה: לבחון פעם אחת ואז לתת לרוץ
  1. בחרו משימה שחוזרת על עצמה ודורשת דפדפן
  2. הפעילו את הסוכן פעם אחת בפיקוח — עקבו אחרי כל צעד
  3. ראו שהוא מסוגל לרוץ מקצה לקצה
  4. ארזו את ההנחיות כסקיל שמנחה אותו בצעדים
  5. בפעם הבאה — שולחים ומסיימים. הוא עושה, אתם ממשיכים

7
🔁 עקרון תיקון הסקיל תוך כדי תנועה
קונספט

סקיל לא נבנה פעם אחת ונשאר כך לנצח. הוא מתדייק תוך כדי שעובדים איתו.

  • שיחה → גילוי שהסוכן יכול לעזור → אריזה כסקיל
  • שימוש → גילוי שמשהו לא מדויק → תיקון
  • שוב שימוש → שוב שיפור → הסקיל מתבגר
# כשמגלים שהסקיל צריך שיפור:
תעדכן את הסקיל הזה על סמך מה שלמדנו עכשיו
💡 טיפ — קוד קצר ב-CLAUDE.md

אפשר להגדיר ב-CLAUDE.md שלכם קוד קצר שיעשה את זה אוטומטית. אצל אביץ הוא כתב TSK — וכשהוא כותב את זה, הסוכן מעדכן את הסקיל הרלוונטי מהשיחה. כל אחד יכול להגדיר קיצור כזה בסגנון שלו.

💡 טיפ — סקיל שנטען בצעדים (Lazy Loading)

כשסקיל גדול — לא כדאי לדחוף הכל לתוך קובץ אחד. הסוכן יטען הכל לזיכרון בבת אחת ויבזבז חלון הקשר יקר. הפתרון: לפרק לקבצי עזר שנטענים לפי צורך.

במקום סקיל ענק אחד:
my-skill/
├── SKILL.md ← תיאור קצר + הוראות ראשיות
├── step-1-research.md ← נטען רק בשלב המחקר
├── step-2-write.md ← נטען רק בשלב הכתיבה
└── examples.md ← נטען רק אם צריך דוגמאות

ב-SKILL.md כותבים: "לשלב המחקר — קרא את step-1-research.md". הסוכן יקרא את הקובץ רק כשהגיע לשלב הזה — לא לפני. חלון הקשר נשאר נקי עד שצריך.

🔨
⚡ תרגיל: התקנה + חיבור + בנייה
HANDS-ON

כולם מתקינים את התוסף ומנסים תהליך אמיתי עם הדפדפן:

  • שלב 1: התקנת Chrome Extension  ← להורדה מ-Chrome Web Store
  • שלב 2: חיבור לחשבון Anthropic בתוסף
  • שלב 3: claude --chrome → הקלידו /chrome לבדיקת חיבור
  • שלב 4: בחרו תהליך שרציתם לעשות ולא ידעתם איך — תנו לסוכן לעשות אותו
  • שלב 5: אמרו לסוכן לארוז את מה שעשינו כסקיל
# אחרי שהסוכן ביצע את התהליך:
תארוז את מה שעשינו עכשיו כסקיל
# להפעיל תהליך ספציפי:
תבצע את התהליך הזה: [תיאור] ואז תארוז אותו כסקיל
🏠
🎯 שיעורי בית
למפגש הבא

מצאו תהליך אחד שדורש דפדפן ועשו אותו עם הסוכן. ואז — ארזו אותו כסקיל:

  • להתקין את תוסף הכרום אם לא הספקתם במפגש
  • לבחור תהליך אחד שחוזר על עצמו ודורש דפדפן
  • לתת לסוכן לעשות אותו ואז לאמר: "תארוז את זה כסקיל"
  • לנסות את הסקיל שוב — לדייק אם צריך (TSK)
